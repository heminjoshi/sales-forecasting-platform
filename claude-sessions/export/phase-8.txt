     Approach

     Three buckets of work, partitioned by file ownership so they can be built

     Foundation (shared/contended files — built first, sequential)

     PostgreSQLContainer<>("postgres:16")
     and a Redis container, each wired via @ServiceConnection (the same Boot
     mechanism
     TopCategoriesReadIT already uses for Postgres). Redis is a
     GenericContainer<>("redis:7").withExposedPorts(6379) +
     @ServiceConnection(name = "redis").
     Why a shared base: both the cache ITs and the forecast/degradation ITs boot
     the full app and
     need both containers; one base avoids each test re-declaring containers and
     lets the parallel
     streams extend it without touching each other.
       - Spring idiom note: @ServiceConnection auto-derives spring.data.redis.*
     /
     spring.datasource.* from the running containers — no @DynamicPropertySource
     needed.
